## 它解决什么问题

你写过不少 Skill。写的时候都挺满意，然后呢？

它在实战里到底表现怎么样——哪几次该触发没触发、哪几次输出被你当场改掉、
哪句话你每次用它都要手动补一遍——**没人记得。你不记得，它自己更不记得。**
于是每个 Skill 都停在"写完那天"的水平：你以为它能用，其实它只是没被验过。

Muninn 是奥丁的记忆渡鸦：每天飞遍世界，黄昏带回真实发生过的事。
这个 Skill 对你的 Skill 库做同样的事——

1. **会话考古**：飞进本地会话记录（transcript，只读），带回每个 Skill 被真实使用的痕迹：
   命中、漏触发、误触发、当场纠正、重复咒语五类证据，每条带坐标。
2. **定级**：Lv1 雏鸦 → Lv5 神使，逐级核验、每格要证据，不凭印象发段位。
3. **铸题**：每次真实失败都变成一道回归考题，只增不删、难度只升不降。
4. **试炼**：一次只练一门功，改完立刻考，考过才升级；红题挂着就冻级，连续不修就降级。
5. **记萨迦**：每个 Skill 一份持续维护的成长档案，下次接着练，不从零开始。

升级不是单看测试绿不绿。每轮试炼**先拍底片、设回刀点**再动刀，升级要过**四道闸**
（核验项 + 考题全绿 + 能力清单逐项仍在 + 底片无退化）——专治"优化着优化着，
某个好功能在产物里悄悄没了，而测试全绿"这类假升级。改错了？回刀点一键还原。

## 五个等级

| 等级 | 名号 | 标准 |
|---|---|---|
| Lv1 | 雏鸦 | 装得上、触发过、核心流程完整跑通过一次 |
| Lv2 | 离巢 | 触发可靠：8 条体检话术全对，实战无漏触发/误触发 |
| Lv3 | 历战 | 失败有备：已知失败模式有应对，危险动作有禁区 |
| Lv4 | 英灵 | 考题傍身：≥5 道源自真实失败的考题，全绿 |
| Lv5 | 神使 | 可出门：陌生人能装上、看懂、三分钟跑出第一个结果 |

升级靠考试，不靠文笔；多数自用 Skill 练到 Lv3-Lv4 就是最优解——
Muninn 不会劝你给一个自用 Skill 补发布材料。

## 它会交付什么

| 产物 | 说明 | 去向 |
|---|---|---|
| **saga（萨迦）** | 成长档案：等级、经验值、升级史、战斗日志、守夜清单 | 永远本地（`~/.claude/muninn/`） |
| **evals（考题集）** | 回归考题，每道有真实出处 | 本地；脱敏后可随你的 Skill 仓库入库 |
| **runestone（符文石）** | 等级卡，可截图 | 会话内输出 |

## 快速开始

```bash
npx skills add uaandesign/muninn
```

装完对 Agent 说：

```text
用 muninn 盘点一下我的 skill 库，告诉我哪个最值得先练
```

## 触发方式

- 「给这个 skill 练级 / 升级我的 skill / skill lvup」
- 「我的 xx skill 实战表现怎么样？」
- 「挖一下这个 skill 的使用记录 / skill 考古」
- 「给 skill 定个级 / 建个评测集」
- 「盘点我的 skill 库，哪个最该修？」
- 「放飞渡鸦」

## 它和同类有什么不同

| 维度 | 常见做法 | Muninn |
|---|---|---|
| 证据来源 | 读仓库文件、当场跑测试 | **先挖真实会话历史**：你实际怎么用、它实际怎么挂 |
| 改进形态 | 一次性优化，产出一份报告 | **持续练级**：成长档案常驻，下次接着练 |
| 评测集 | 临时设计测试 prompt | **失败铸题**：每次真实失败永久成为回归考题 |
| 防退化 | 测试绿了就算改好 | **能力清单 + 出厂底片**：没失败过的好功能也受保护，删了/产物变差就回刀 |
| 可回滚 | 改完才发现错，难复原 | **回刀点**：每轮试炼一个可寻址检查点，还原命令记进档案 |
| 质量方向 | 只升不降 | **会掉级**：红题冻级、能力流失冻级、连冻降级，逆水行舟 |
| 范围 | 单个 Skill | 顺带**全库盘点**：用得多 × 等级低 = 优先练 |

> **升级要过四道闸**：核验项通过 + 考题全绿 + 能力清单逐项仍在 + 出厂底片无退化。
> 任一不满足就回刀——"测试绿了"只是其中一道，绿灯下能力静默缩水是头号假升级。

## 安全边界

- **会话记录只读**：绝不修改、移动、删除任何 transcript。
- **记忆留在巢里**：含会话引用的 saga 和战斗日志永远在本地；
  进入任何公开产物（考题入库、截图、commit）的内容必须先按脱敏规则清洗。
- **碰你的文件前先问**：修改你的 SKILL.md、把考题写进你的仓库、一切 push/发版动作，
  每次都需要你明确的指令；疑问句不算指令。
- **不伪造**：空手而归如实报空，预估分数明确标注"预估"。

## 验证与测试

装完后说：「用 muninn 给 <你的某个 skill> 定个级」。

合格的表现是：先跑会话考古并给出**带坐标的证据清单**（或如实报告无实战记录），
再做触发体检，然后输出逐级核验的定级表和符文石——而不是跳过证据直接开始改文件。
