# 实战案例：能力清单拦住了一次"测试全绿的假升级"

> 对象：一个自用的「AI 安全周报」Skill（检索 → 硬校验 → 飞书卡片 → 归档）。
> 时间：2026-06-14。本案例脱敏，只保留 Muninn 机制相关的数字与判定。
> 它回答一个真问题：**"优化"之后某个好功能在产物里悄悄没了，而所有测试还是绿的，怎么办？**

## 背景：一个补丁修不到的盲区

这个周报 Skill 之前被升级过一轮（修了校验时钟、触发词、免责声明），考题全绿、
内置的 `validate.py` 18 项全 PASS。看起来是个干净的升级。

但物主发现：**升级前周报里的新闻是带配图的，升级后不带了。**

排查发现配图功能**根本没被删**——渲染逻辑、配图规则、validate 的配图检查全在。
真正发生的是：生成某一期时跳过了"打开原文抠图"这步体力活，把图字段全留空了；
而 Skill 的规则恰好允许"无图就留空，是正常形态"，于是 validate 照样 PASS。

> 这就是考题机制的盲区：考题全从**失败**长出来，可配图功能从没"失败"过，
> 没有任何考题盯着它。它在产物里静默缩水，每一道闸都放行。

## Muninn 怎么接住的

### 1. 盘能力，冻清单

```bash
bash tools/capability-manifest.sh <skill目录> ~/.claude/muninn/<skill>/capability-manifest.md
```

工具自动扫出章节、产物字段、脚本、外部调用。人工补全时，把"新闻配图"列为一项 **C4**，
并给它写明**判定标准**（关键就在这句）：

> C4 新闻配图：`render_card.py` 仍读 `cover_img_key` 并渲染；
> **且最近一期产物里 ≥1 条新闻带图**。
> （有合格图才配是能力，全空是退化、不是正常形态。）

注意判定标准没停在"代码在不在"，而是落到"**真实产物里出没出现**"——
这正是补丁视角和产物视角的区别。

### 2. 核对：同一份数据，两个判定

对当期产物（5 条新闻配图全空）核对：

| 谁在判 | 判定 | 结论 |
|---|---|---|
| `validate.py` 配图检查 | **PASS** | 留空是允许的 |
| Muninn 能力核对 C4 | **FAIL 🔴** | 产物 0/5 带图（上一期 3/5），能力在产物里消失 |

绿灯和红灯指着同一份数据。validate 问"格式合不合规"，能力清单问"那个功能还在不在"。

### 3. 冻级，不放行

按 Muninn 升级四道闸（核验项 + 考题全绿 + **能力清单逐项仍在** + 底片无退化），
C4 消失 → **能力流失冻级⚠️**。这一期本来是冲 Lv4 的"主考"，结果不是升级，是冻结。

> 结果是诚实的：跑通了全流程 ≠ 升级成功。一个好功能在产物里没了，
> 哪怕 validate 全绿，也不算练强，算练弱了。

## 4. 解冻：两轮试炼，一轮一门

冻级不是终点。按"留底片 → 设回刀点 → 改一门 → 过验证门 → 核能力"各走一轮：

**试炼 A — 补图重发**：抠 4 张原文配图（不含 logo/广告，≥600px）上传，产物从 0/5 升到
4/5 带图。底片对比：`img_key` 数 0 → 4，无字段丢失（只增图，没顺手改坏别的）。
能力核对 C4：🔴 → 🟢 解冻。卡片重发飞书。

> 中途插曲：改完日期还是 6/12 的旧值，**被另一道能力闸 C7「报告日期为当天」拦下**
> （6/14 发 6/12 的报告 → FAIL）。改成当天才放行。两道闸彼此兜底，这才是清单的价值。

**试炼 B — 治本**：把教训沉淀成工具里的闸。validate 新增 `_check_cover_count` 软门槛——
Top5 带图 <3 时 **WARN**（不阻断，守住"宁缺毋滥"），提醒"是否漏抠了图"。
实测：4 张图静默放行，0 张触发 `[WARN] 仅 0/5 条带图`。这道闸若早存在，
当初那次 0 图就会被当场提醒，而不是溜到产物里。

## 5. 结果：跑通 → 冻级 → 修复 → 晋级

这条完整路径正是四道闸该有的样子：

| 阶段 | 等级 | 触发 |
|---|---|---|
| 实跑第13期，validate 18 项全过 | Lv3（曾误判为"主考通过"） | 流程跑通 |
| 能力核对发现 C4 退化 | Lv3 **冻结⚠️** | 能力流失冻级 |
| 试炼 A 补图解冻 + 试炼 B 治本 | — | C4🟢、C10 入册 |
| 四道闸全过（核验项+考题+能力+底片） | **Lv4 英灵** | 真升级 |

**跑通流程 ≠ 升级。** 一份测试全绿的产物，能力清单照样能把它摁住，直到好功能真的回来。

## 留下的资产

- **能力清单**：10 项受保护能力（含新加的 C10 配图软门槛），每项带"怎么确认还活着"。
- **回刀点**：补图前快照 + 加门槛前的 validate 备份，连还原命令记进 saga——改错了一键回退。
  （顺带验证了"回刀点别放 /tmp"：上一轮存在 /tmp 的基线，重启后真的没了。）
- **工具里的新闸**：`_check_cover_count` 软门槛，让这次教训长在 validate 里，不只活在案例中。

## 一句话

考题防"已知的坏复发"，能力清单防"已有的好流失"——这次是后者救了场；
而修复的终点不是"改回来"，是把教训拧成工具里的一道新闸。
