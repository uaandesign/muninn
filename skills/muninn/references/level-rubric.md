# 等级量表（Level Rubric）

> 定级的唯一依据。逐级核验，不跳级认定；每一格都要证据坐标（考古记录 / 对练记录 / 文件路径）。
> 升级 = 该级全部核验项通过 **且** 全部考题绿。经验值不能抵考试。

## Lv1 雏鸦 · 能跑

**标准：装得上、触发过、核心流程完整跑通过一次。**

| 核验项 | 怎么验 |
|---|---|
| 结构完整 | SKILL.md 存在，frontmatter 有 `name` + `description` |
| 触发过至少一次 | 考古找到任意一次触发记录；或当场用一条话术实跑触发 |
| 核心流程完整 | 找到（或当场跑出）一次从触发到产出的完整记录，没有中途崩掉 |

**试炼（升 Lv2 的考试）**：触发体检 8 条话术（5 正 3 负）全对。

**典型卡级原因**：description 只写了"做什么"没写"什么时候用"；触发词全是开发者术语，
用户的自然说法一条都接不住。

---

## Lv2 离巢 · 触发可靠

**标准：用户怎么自然地说，它就怎么准确地醒。**

| 核验项 | 怎么验 |
|---|---|
| 体检全对 | 8 条话术（5 正含 2 条不带 skill 名的自然说法 + 3 负）全部判断正确 |
| 实战无漏触发 | 考古窗口内没有"用户描述了场景但它没醒"的记录 |
| 实战无误触发 | 考古窗口内没有"它醒了但用户说不是要这个"的记录 |
| 负触发写明 | description 里有"不要用于…"，至少挡住 2 个相邻场景 |

**试炼（升 Lv3 的考试）**：对抗对练——设计 3 条故意制造麻烦的输入
（缺前置条件 / 输入格式不对 / 介于触发与不触发之间的模糊场景），
Skill 的表现必须是"明确处理或明确拒绝"，不能是静默出错或装作正常。

**典型卡级原因**：触发词只覆盖"功能词"，没覆盖"人话"；
没有负触发，把隔壁 Skill 的活也抢了。

---

## Lv3 历战 · 失败有备

**标准：已知的失败方式都有应对，危险的动作都有禁区。**

| 核验项 | 怎么验 |
|---|---|
| 失败模式编码 | SKILL.md 写明 ≥3 种已知失败场景及对应处理（fallback / 标注 / 停手） |
| 危险动作有禁区 | 涉及删文件、shell、git、发消息、外部 API 的，有明确的"不做什么"清单 |
| 停手点存在 | 不可逆/对外动作前会停下来等用户确认 |
| 对抗对练通过 | Lv2 试炼的 3 条对抗输入全部"明确处理或明确拒绝" |

**试炼（升 Lv4 的考试）**：考题集建成——≥5 道考题，每道都有真实出处
（实战失败 / 体检失误 / 对抗对练），全部绿。

**典型卡级原因**：失败处理写的是"视情况灵活处理"这类没法执行的话；
危险动作的边界只存在于作者脑子里。

---

## Lv4 英灵 · 考题傍身

**标准：它的每一次跌倒都变成了考题，而它现在全都考得过。**

（英灵殿的战士每天战斗、倒下、复生，越战越强——这一级的 Skill 同理：
失败不可耻，失败没有变成考题才可耻。）

| 核验项 | 怎么验 |
|---|---|
| 考题 ≥5 且全绿 | 逐题重跑，记录在案 |
| 考题有出处 | 每道题标明来自哪次真实失败/体检，没有凭空编的题 |
| saga 连续 | 升级史、战斗日志、守夜清单齐全且最近一次出动有更新 |
| 无冻级标记 | 没有红题挂着 |

**试炼（升 Lv5 的考试）**：陌生人测试——切换到从没见过这个 Skill 的用户视角，
只看仓库（不看 saga），回答：这是干嘛的？怎么装？装完第一句话说什么？跑完我能看到什么？
四问都能在 3 分钟内从仓库本身找到答案，才算过。

**典型卡级原因**：考题全绿但都是"自家人出的送分题"——没有一道来自真实失败。

---

## Lv5 神使 · 可出门

**标准：能被一个陌生人装上、看懂、三分钟跑出第一个结果。**

只有用户明确想发布时才核验这一级。**多数自用 Skill 的最优解是 Lv3-Lv4，
不要劝用户为自用 Skill 补发布材料。**

| 核验项 | 怎么验 |
|---|---|
| 陌生人四问通过 | 见 Lv4 试炼 |
| 一行安装实测 | 安装命令真跑一遍，贴输出 |
| 产物可见 | README 里有真实运行产物（截图/GIF/输出样例），不是虚构样例 |
| 安全边界公示 | README 写明不做什么、何时停手 |
| 考题随仓库 | 脱敏考题以 `evals/` 入库，外人可以自己验 |
| 无隐私泄漏 | 全仓库无 key、token、私人路径、会话原文 |

---

## 降级与冻结

- **红题冻级**：任何考题变红 → 等级冻结⚠️，修复全绿后解除。
- **连冻降级**：连续两次出动未修复 → 降一级，saga 记明原因。
- **降级不删史**：升级史保留，降级作为新条目追加——萨迦记录全部历史，不洗白。

## 经验值

- 1 条有效证据（带坐标）= 1 点经验；超过 90 天的证据折半。
- 经验值只用于全库盘点时排培养优先级（用得多且等级低的优先练）。
- 经验值不参与升级判定。**升级只看核验项和考题。**
