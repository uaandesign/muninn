# 能力清单指南（Capability Guide）

> 怎么盘、怎么补、怎么核——守住"没失败过的好功能"。

## 为什么需要它

考题是**事后**机制：先有失败，才有考题。可一个 Skill 里大量功能**从没失败过**，
于是没有任何考题盯着它们。试炼时把它们顺手删掉、或让产物里悄悄不再出现，
所有考题依然全绿，validate 依然 PASS——判出一个**假升级**。

> 真实教训：某周报 Skill"优化"后，新闻配图在产物里消失（5 条全空），
> 而 `validate.py` 全程 PASS——因为规则允许"无图就留空，是正常形态"。
> 绿灯下能力静默缩水，没有一道闸拦住。

能力清单就是给每个好功能发一张护身符：**消失即回刀，与考题无关。**
它和考题互补——考题防"已知的坏复发"，清单防"已有的好流失"。

## 一、盘点：跑工具拿初稿

```bash
bash tools/capability-manifest.sh <skill目录> ~/.claude/muninn/<skill>/capability-manifest.md
```

工具自动扫五段：
- **A 声明的能力**：SKILL.md 的章节标题
- **B 产物字段/渲染点**：img_key / cover / payload / render / archive 等信号
- **C 脚本与依赖**：每个脚本是一类能力
- **D 外部命令/API**：lark-cli / curl / `/open-apis/...` / requests 等
- **E 人工补充**：留空，给你填自动扫不到的隐性能力

## 二、补全：把初稿改成可核验的清单

自动初稿**不能直接用**，必须人工过一遍：

1. **删误报**：A 段会混进"参考""正反例"这种非能力章节，删掉。
2. **补隐性能力（最重要）**：扫描器只看得见"字段和脚本"，看不见"质量承诺"。例如：
   - "宁缺毋滥的配图"——*有合格图才配* 是能力，*全空* 是退化，不是正常形态。
   - "免责声明每期都在"——少了就是合规缺口。
   - "战略洞察结构化成方向卡"——退回大段文字墙就是退化。
   这些写进 E 段。
3. **给每项写"怎么确认它还活着"**——一句**可执行**的检查，不能是"看起来在"：

   | 能力 | 怎么确认还活着（代码位置 + 产物判断） |
   |---|---|
   | 新闻配图 | `render_card.py` 仍读 `cover_img_key`；且最近一期产物里 ≥1 条新闻带图 |
   | 发送后归档 | `--archive-confirmed` 路径仍在；产物 `card_payload` 含往期链接 |
   | 免责声明 | validate 有免责声明检查；渲染产物两个板块结尾各有一条 |

   判断要落在**真实产物**上，不是只看代码在不在——代码在、产物里却不出现，
   就是这次事故的形态。

## 三、冻结与连续

- **首次盘点即冻结**：存 `~/.claude/muninn/<skill>/capability-manifest.md`，作为基线。
- **已有清单读旧的**：只增补（发现新隐性能力就加），不重盘——和 saga 一样靠连续。
- **清单只增不删**：确实废弃的能力，移到"已废弃"段并写明"谁、哪轮、为什么授权删"，
  不许在优化里静默蒸发。

## 四、核对：每轮试炼收尾逐项过

照清单里每项的"怎么确认还活着"去**真实产物**里看，结果记进清单末尾的核对表：

```markdown
| 日期 | 试炼 | 全项仍在? | 消失项 | 处置 |
|---|---|---|---|---|
| 2026-06-14 | 修触发词 | 是 | — | — |
| 2026-06-14 | 重构渲染 | 否 | 新闻配图（产物 0/5 带图） | 回刀，还原渲染逻辑 |
```

**任何一项从有变无 = 本轮回刀**，哪怕考题全绿、validate 全 PASS。
若该能力确实该废弃 → 停手等用户**指令句授权** → 授权后移入"已废弃"段并记录。

## 五、和底片的分工

- **能力清单**回答"**哪些**功能必须还在"（离散的能力点，逐项打勾）。
- **底片对比**回答"产物**整体**有没有变差"（同输入跑前后两份产物，逐项 diff，
  抓字段缺失、篇幅塌缩、数量下降等清单可能没列到的退化）。

两个一起用：清单守已知的能力点，底片兜清单的漏网之鱼。
