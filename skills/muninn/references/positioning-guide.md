# 定位指南（Positioning Guide）

> "值不值得练"这一步的详细清单。挖完使用记录之后、定级之前做一次，
> 结论冻进 `~/.claude/muninn/<skill>/positioning.md`，以后出动只读不重做
> （除非用户要求、或前提信号明显变了）。
>
> 一句话目的：**别对着一个不该存在的 Skill 精修。** 大部分 Skill 的真问题不是写得烂，
> 是没想清楚为什么值得装。

## A. 验前提（这个 Skill 凭什么存在）

逐条作答，先给结论再展开：

1. **真实问题**：它解决的问题真的成立吗？还是一个"听起来有用"但没人真会遇到的问题？
2. **安装理由**：用户为什么要装它，而不是临时问一句 Agent 就完事？
   （能被一句 prompt 替代的，就不值得做成 Skill。）
3. **独特资产**：它的唯一性来自哪——方法论 / 脚本资产 / 数据 / 工作流 / 展示效果？
   一样都数不出来，就直说"同质化风险高"。
4. **传播钩子**：有没有一句话能讲出去、让人想转发或想装的钩子？
5. **接会话考古**（Muninn 特有的硬证据）：如果连作者自己都很少用、触发词从没被自然
   说出口过——**这是前提不成立的最强信号**，比任何主观论证都重。

> **结论三选一**：
> - **[值得练]**：前提成立，进入正常流程。
> - **[值得，但要先调定位]**：问题成立但站位不对（如把消费侧工具写成维护侧触发），
>   先定位再练。
> - **[前提存疑]**：问题不成立 / 无独特性 / 没人用。**停手**，把判断和理由抛给用户，
>   给"重想定位 / 砍掉 / 仍坚持"三个选项，不要自己接着定级改文案。

## B. 探行情（横向找同类，不闭门造车）

**必须联网**，不能只凭已有知识。找相近的东西，分三类，**每个带真实链接**：

| 类型 | 找什么 |
|---|---|
| 直接竞品 | 解决同一个问题的 Skill / 工具 |
| 间接竞品 | 解决相邻问题、用户可能二选一的 |
| 手艺标杆 | 不一定同类，但 README / 命名 / showcase / 安装体验做得特别好，值得偷师 |

去哪找：GitHub、ClawHub、skills.sh、Tessl，以及搜索引擎。搜索词从这个 Skill 的
name、description、核心任务里抽，分三组：功能词（它做什么）、人群词（谁会用）、
形态词（skill / agent / runtime 名）。

工具纪律：优先用 `curl` / `gh` 这类不弹权限框的 CLI；WebFetch / WebSearch 可能触发
用户看不见的权限弹窗导致静默挂起，连续失败就换路子，别原地干等。**找不到就如实说
"搜了哪些词、哪些渠道没结果"，绝不凭印象编"有些项目"。**

对标表：

```markdown
| 同类 | 链接 | 类型 | 一句话定位 | 它凭什么被理解/装/传播 | 可偷的手艺 |
|---|---|---|---|---|---|
```

## C. 定位置（纵向来路 + 横向立足点 → 一句话新定位）

### 纵向：它从哪来、要去哪

- 最初为解决什么**具体痛点**？
- 现在是哪种形态：工具 / 方法论 / 工作流 / 风格迁移 / 自动化系统？
- 从"私用"到"公开可用"还**缺哪一步**？
- 下一版最该往哪条路走：更强功能 / 更好展示 / 更稳安装 / 更通用适配 / 更高验证？

### 横向：同类凭什么立足（至少过这几条）

| 维度 | 自问 |
|---|---|
| 命名钩子 | 名字有记忆点吗？一听知道解决什么吗？ |
| 一句话定位 | 用人话说清用途了吗？ |
| 安装摩擦 | 一条命令能装吗？要复杂前置条件吗？ |
| 首屏信任 | README 首屏有徽章 / GIF / 截图 / 结果样例 / 真实数据吗？ |
| 可验证产物 | 跑完有 HTML / PDF / 报告 / 卡片 / diff / 测试结果这种看得见的东西吗？ |
| 安全边界 | 写明不乱删 / 不泄露 / 不擅自发外部请求了吗？ |
| 生态兼容 | 明确兼容多个 Agent runtime 吗？ |
| 故事感 | 在讲"为什么现在需要它"，还是只列功能？ |

### 交叉 → 一句话新定位

把纵向的"该往哪走"和横向的"同类靠什么立足"交叉，得出**该抢的生态位**，
凝成一句话。这句话会喂给后面的试炼，指挥"下一版优先练哪条路"。

> 横向这些维度里，大量是**主观或依赖业务背景**的判断（命名好不好、问题对你的用户成不
> 成立、该不该改定位）。这些一律按 SKILL.md 的铁律**抛给用户**，带选项问，别自己拍板。

## positioning.md 落盘格式

```markdown
# <skill> · 定位卡（首次冻结 YYYY-MM-DD）

## 验前提结论：[值得练 / 值得但调定位 / 前提存疑]
- 真实问题：…
- 安装理由：…
- 独特资产：…
- 传播钩子：…
- 会话考古佐证：…

## 同类对标（带链接）
| 同类 | 链接 | 类型 | … |

## 定位
- 纵向（来路→去向）：…
- 横向（立足点）：…
- **一句话新定位**：…

## 抛给用户待决（主观/业务判断）
- [ ] …（带"我的建议 + 备选 + 为什么"）
```
